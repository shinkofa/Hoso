### D. Fix bug bloquant — image noire au premier stream (fork StreamPack)

**Symptôme** : 1er démarrage = stream connecté mais **image noire** toute la session ;
2e essai = image OK. Rédhibitoire pour un testeur (abandon avant le 2e essai).

**Méthode (LOGS FIRST)** : 2 captures logcat sur device (CPH2173), dont un run propre
sans interaction (2min20). Écarté successivement : course encodeur audio (faux — c'était
le teardown), kill OEM de l'AudioRecord (faux — run propre a tenu 2min20), interaction
utilisateur. **Cause racine prouvée** : le premier `VirtualDisplay` AUTO_MIRROR, créé
juste après le consentement MediaProjection, miroite du **noir** toute la session (le
miroir se lie au layerStack du display par défaut un instant trop tard sur ColorOS/A14) ;
le 2e essai marche car le lien projection est déjà chaud.

**Correctif** : délai de mise en place de **600 ms** avant `createVirtualDisplay` dans
`MediaProjectionVideoSource.kt`. Fork compilé en local (`includeBuild`). **Vérifié sur
device** : 1er essai affiche désormais l'écran, démarrage < 12 s. Commit fork `f14e0ae`
(branche `fix/twitch-audio-race-hoso`, poussé sur theermite/StreamPack).
