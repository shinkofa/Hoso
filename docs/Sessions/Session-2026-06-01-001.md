## Erreurs et corrections

| Erreur | Cause | Correction |
|--------|-------|------------|
| `adb` not in PATH | SDK platform-tools pas dans PATH system | Chemin absolu |
| JAVA_HOME not set | Pas d'export par défaut | `export JAVA_HOME` avant gradlew |
| keytool PKCS12 password constraint | store_pwd ≠ key_pwd non supporté en PKCS12 | Recréation avec pwds identiques |
| Shinkofa-Vault identifié comme HashiCorp | Mauvaise hypothèse initiale | Jay a corrigé → SOPS+age sur VPS |
| SOPS "no matching creation rules" | Chemin `/tmp/` ne match pas la regex `secrets/.*\.yaml` | Écriture directe dans `secrets/` + `--encrypt --in-place` |
| generate-envs.sh "Key not found" | SSH non-interactif sans `~/.local/bin` dans PATH | `bash -l -c` (login shell) |
| Gradle keystore path Windows | Git Bash `/c/Users/...` traité comme relatif par Gradle | `cygpath -w` dans fetch-signing.sh |

## Décisions

| Décision | Raison |
|----------|--------|
| Apache 2.0 | Compatible StreamPack upstream (aussi Apache 2.0) |
| Shinkofa-Vault comme source of truth signing | Centralisé, chiffré, pas de secrets en clair |
| Fallback debug signing sans Vault | Contributeurs open-source buildent sans accès Vault |
| `@Volatile isRunning` static flag | Plus simple et fiable que query intent pour le guard overlay |
