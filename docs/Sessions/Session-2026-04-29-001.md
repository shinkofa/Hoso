### Insight critique identifié

Le bug "Mix produisant du silence" du cycle 6 s'explique par la convention raw layout :
- Les buffers StreamPack sont en `position=0, limit=capacity`.
- `AudioRecord.read(direct ByteBuffer, ...)` ne bouge PAS la position (cf javadoc Android), donc post-read l'état reste raw layout.
- Le code cycle 6 a appelé `flip()` sur des buffers en `position=0` → `flip()` met `limit=position=0` → `remaining()=0` → boucle de mix = 0 itération → output vide → silence.

**Règle gravée** : ne jamais appeler `flip()` dans le pipeline audio StreamPack.
