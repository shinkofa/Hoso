### F2 — Cause racine MIC silencieux identifiée

**Logcat révèle** :
```
AudioRecord: set(): inputSource 5, sampleRate 44100, ...
AudioFlingerExtImpl: oplusSetParameters: keyvalue CamcorderSwitchMic=false
```

`inputSource 5` = `MediaRecorder.AudioSource.CAMCORDER` (pas MIC=1).

**Source StreamPack 3.1.2** (`MicrophoneSource.kt:71`) :
```kotlin
class MicrophoneSourceFactory(
    @AudioSourceValue val audioSource: Int = MediaRecorder.AudioSource.CAMCORDER,
    ...
)
```

→ Le default StreamPack est CAMCORDER. Notre code `MicrophoneSourceFactory()` (sans arg) hérite ce default.

**Hypothèse** : ColorOS gère CAMCORDER comme source synchronisée avec caméra → en absence de caméra active, audio routé vers mic arrière inactif ou muté (`CamcorderSwitchMic=false`).

### F3 — Cause restart bug identifiée (partiellement)

`OverlayService.kt:115-122` (bouton stop overlay) déclenche STOP via broadcast.
`MainActivity.stoppedReceiver` (lignes 45-55) reset `streamer=null, connection=null` **mais ne fait PAS de `unbindService`** (contrairement à `stopStreaming()` ligne 266).

→ ServiceConnection côté Activity reste bindée à un service mort après stop overlay.
