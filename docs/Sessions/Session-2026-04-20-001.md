## Travail réalisé

### Hōsō — Stream RTMP fonctionnel sur Twitch

1. **Correction VirtualDisplay orientation** — root cause du "stream minuscule dans le canvas Twitch" : VirtualDisplay capturait selon l'orientation initiale du téléphone. Fix appliqué dans `MainActivity.kt` : forcer `ActivityInfo.SCREEN_ORIENTATION_LANDSCAPE` avant la capture MediaProjection. Restaurer `SCREEN_ORIENTATION_UNSPECIFIED` au stop.
2. **AndroidManifest** : ajout `configChanges="orientation|screenSize|screenLayout|smallestScreenSize"` sur MainActivity → évite la recréation lors de la rotation.
3. **Presets résolution Fit/Crop auto-calculés** (`StreamConfig.kt`) :
   - Fit mode : encode à l'aspect ratio natif du téléphone (1920x864 pour 20:9), Twitch ajoute des bandes noires haut/bas (pas de rognage).
   - Crop mode : encode en 16:9 strict (1920x1080), bords gauche/droite rognés.
   - `buildPresets(native)` dérive la hauteur depuis le ratio réel du téléphone.
   - Clé SharedPreferences bumped à `res_idx_v3` pour invalider les index sauvegardés périmés.
4. **Swap largeur/hauteur dans `configureAndStart`** : force `maxOf(w,h) × minOf(w,h)` pour garantir dimensions landscape côté encoder même si le raw preset a été stocké en portrait.
5. **Repo GitHub créé** : https://github.com/shinkofa/Hoso (privé, organisation Shinkofa). Commit `feat(stream): working RTMP stream with overlay, Fit/Crop resolution modes` poussé sur `main`.
