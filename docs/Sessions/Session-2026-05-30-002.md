## Travail réalisé

### Batch 1 — Audio backend (`9b15518`)

- `AudioGains` : singleton `AtomicInteger` permil (0..2000, 1000=100%).
  Lock-free, allocation-free. Lu par le thread audio à chaque frame,
  écrit depuis les sliders UI.
- `MixedAudioSource` : composite `IAudioSourceInternal`. Le micro
  fournit l'autorité timestamp via `getAudioFrame`. Le jeu est lu dans
  un scratch buffer direct réutilisable (native byte order). Mix in
  place par indexation absolue, sans mutation de position. Clipping à
  ±32767.
- `MixedAudioSourceFactory` : instancie `MicrophoneSourceFactory` +
  `MediaProjectionAudioSourceFactory`, les passe au mixer.
- `AudioSource` enum : MIC / MIX. Clé legacy `"game"` migrée vers MIX.
- `StreamConfig` : deux nouvelles prefs (`mic_gain_permil`,
  `game_gain_permil`). `init` hydrate `AudioGains` depuis prefs. Setters
  persistent ET synchronisent `AudioGains` en une écriture.

### Batch 2 — UI principale (`4fe55a5`)

- `group_mix_gains` (gone par défaut) sous le spinner, visible en MIX.

### Bug fix — `position()` → `capacity()` (folded dans Batch 1)

**Symptôme observé par Jay** : sliders visibles, mais aucun effet
audible. Mic et son du jeu restent au même niveau quels que soient
les sliders, même à 0 %.

**Cause racine** : `AudioRecord.read(ByteBuffer, sizeInBytes)` écrit
les bytes PCM dans le direct buffer mais **ne déplace PAS `position()`**
(comportement Android documenté). `mixInPlace` calculait
`bytes = minOf(mic.position(), game.position())` → 0 sur les deux
buffers → boucle de mix jamais exécutée → gain jamais appliqué, et le
jeu jamais mixé dans la frame envoyée à l'encodeur.

**Fix** : utiliser `capacity()` (taille réelle des buffers PCM remplis
par AudioRecord). Bonus : gain mic appliqué même si lecture jeu échoue
(`gameOk=false` → `gGame=0f`, mic gain toujours actif). Validé par
Jay : "Cette fois-ci, ça fonctionne."
