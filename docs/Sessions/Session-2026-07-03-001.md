### A. GlitchTip câblé (commit `9e0b5ee`)

erreurs only. Preuve bout-en-bout : DSN résolu → `monitoring.theermite.com` projet 7,

### B. Durcissement démarrage FGS (commit `0b4aedb`)
Garde autour de `startForeground` (`ScreenRecordService.onCreate` + `StreamPermissionActivity`) :
classifieur `StreamStartError` → capture GlitchTip taguée → toast lisible → arrêt propre au lieu
d'un crash muet. Cible les échecs FGS stricts (Samsung One UI 7 / S25). TDG : 7 tests.

### C. Fix crash Android 10 + gate lint (commit `7f069e2`)
`getMaximumWindowMetrics()` (API 30) appelé sur minSdk 29 → crash au lancement sur Android 10.
Prouvé par lint (4 erreurs `NewApi`). Corrigé via helper `ScreenMetrics` (garde SDK_INT + fallback
legacy). Gate durcie : `NewApi` élevé en fatal → le build release échoue désormais dessus (prouvé :
release échouait avant fix, passait après). Récepteurs migrés vers `ContextCompat.RECEIVER_NOT_EXPORTED`.

### D. Onboarding premier lancement (commit `ca54659`)

Skippable, re-consultable via « Revoir le tutoriel ». Dignity + ND-friendly (≥16sp, contraste,
reduce-motion honoré). Textes en clés i18n. Dép `androidx.viewpager2@1.1.0`. TDG : 5 tests.
