## Travail réalisé

### G1.1 — DestinationPreset + StreamConfig migration (`58dc549`)

- Nouveau modèle `DestinationPreset` (id, name, type enum
  TWITCH/YOUTUBE/KICK/CUSTOM, rtmpUrl, streamKey, resolutionIndex,
  videoBitrate) avec sérialisation JSON via `org.json`.
- `StreamConfig` étendu : liste de présets + `activePresetId`.
  Tous les accesseurs existants (streamKey, rtmpBaseUrl, etc.)
  routent désormais à travers le préset actif.
- Migration silencieuse : au premier lancement, les anciennes prefs
  sont absorbées dans un préset Twitch par défaut sans action
  utilisateur.
- `deletePreset()` garantit qu'il reste toujours au moins un préset
  (re-seed Twitch si le dernier est supprimé).

### G1.2 — UI préset switcher + dialog (`951aced`)

- Spinner "Destination" full-width en haut du layout.
- Row 3 boutons : ＋ Nouveau / Modifier / Supprimer (equal weight).
- Dialog d'édition (`dialog_preset_edit.xml`) : nom, type spinner,
  URL RTMP, stream key. Sur nouveau préset, le type pré-remplit
  URL + nom. Sur préset existant, jamais d'écrasement silencieux.
- Switching de préset sauvegarde les edits en cours dans le préset
  précédent avant de charger le nouveau.
- `suppressPresetSpinnerEvent` évite un phantom-switch au premier
  adapter assignment.

### G2.1 — Overlay auto-transparence (`915e94e`)

- `Handler` + `Runnable` sur le main looper : après 7 s sans touch,
  l'ImageView overlay fade vers alpha 0.3 (400 ms).
- Tout touch (ACTION_DOWN) annule le timer et snap alpha → 1.0
  (120 ms). Timer reprogrammé à ACTION_UP.
- Callback nettoyé dans `onDestroy()`.
